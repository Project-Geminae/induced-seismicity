1. The question we're trying to answer

A regulator looking at an induced seismic event in the Midland Basin wants a number that helps them act. The interesting actions are roughly:

  1. Deny new permits in problematic geology
  2. Cap volumes on existing wells
  3. Order shut-ins after a specific event
  4. Set risk-based monitoring thresholds

The dashboard's per-event interaction (click an event → see contributing wells) is targeted at action #3: "given that this M3.4 just happened, which of these 12 wells are most likely 'responsible' and which are bystanders?"

This is a different question from "what's the average effect of injection on seismicity in the basin?" — that's the population-level ATE that the TMLE shift / dose-response / mediation drivers compute. This page is about the per-event attribution layer that sits on top of the same Q model.

2. Causal estimands and which one we compute

There is no single "the causal question" — there is a hierarchy, and different estimands answer different policy questions:

Estimand Question it answers Have we computed it?
ATE (Average Treatment Effect) "On average across all well-days, what's the effect of injection on expected ML?" Yes — see tmle_shift_*.csv, tmle_dose_response_*.csv
CATE (Conditional ATE) "What's the effect for this specific well given its depth, fault distance, and neighbor wells?" Yes — Causal Forest DML at all 20 radii, with 95% CIs
ITE (Individual Treatment Effect) "For this well on this day, what's the effect?" Not identified from observational data alone
g-computation prediction "What does the model predict for this well's contribution under its actual injection vs. zero injection?" Legacy (build_attribution_q.py) — superseded by Causal Forest CATE above
PN (Probability of Necessity, Pearl 2000) "Given that this event happened with this well injecting, what's the probability it wouldn't have happened with the well at zero?" A simplified fractional-reduction proxy is shown; not the formal Pearl PN
Shapley attribution "What's the unique fair share of credit for each well, given non-additive interactions in the Q model?" Not computed; deferred
TMLE is a method, not an estimand. The TMLE machinery can target any of these — what differs is the unit of analysis, the DAG, and what counterfactual you're contrasting against. The per-event attribution shown in the sidebar is a per-well g-computation prediction, not an identified causal effect.

3. What the dashboard actually computes

The dashboard uses two complementary estimation methods, each targeting a different level of analysis:

3.1. Population-level: TMLE (v3)

Targeted Maximum Likelihood Estimation computes the population-average causal effect of injection on seismicity. The current implementation (v3) includes seven methodological improvements: 

Outcome:     outcome_max_ML  (max ML within R km on this day, or 0)
Treatment:   cum_vol_365d_BBL
Confounders: perf_depth_ft, days_active, Nearest Fault Dist (km),
             Fault Segments ≤ R km, depth_class (shallow/mid/deep),
             neighbor_cum_vol_7km (spatial interference, VIF=1.12)

Q model:     Hurdle Super Learner
             Stage 1: P(Y > 0 | A, L)  logistic stack
             Stage 2: E[log(1+Y) | Y>0, A, L]  regression stack
             Base learners: Ridge, GBM, XGBoost, RandomForest (4 diverse families)
             Meta-learner:  NNLS (non-negative least squares, convex combination)
             Cross-validation: 5-fold

g model:     Histogram conditional density g(A|L)
             Bins: data-adaptive (~140 via Freedman-Diaconis, was 20 fixed)
             Positivity floor: data-adaptive (1% of 2.5th percentile)
             Classifier: XGBoost multinomial

Targeting:   H = g(A_pre|L) / g(A|L) / (1+δ)
             H-truncation at 99th percentile (max H ≈ 5, was 335)
             ε solved via OLS on residual

Variance:    Influence-function SEs with Bessel correction
             for finite number of clusters

Result:      17/20 radii significant at raw α=0.05 (4–20 km)
             Post-multiplicity: 1/19 Bonferroni, 10/19 BH-FDR
             Combined test: pressure band 7–19 km ψ=+2.4e-3, p<10-17
                                near-field 1–6 km null, p=0.08

TMLE estimands: shift intervention (ψ = E[Yd(A)] − E[YA]), dose-response curve (E[Ya*] at grid points), and mediation decomposition (NDE/NIE via Zheng & van der Laan 2012). Python implementation validated against R tlverse to <0.1%.

3.1.1 Estimator comparison at 7 km

Subsequent diagnostic work (April 2026) revealed a methodological tension: full-panel plug-in estimators (OLS, XGBoost, undersmoothed HAL) give point estimates 5–25× smaller than targeting-based estimators (TMLE v3, CV-TMLE). The table below documents this honestly:

3.1.2 Diagnostic findings

Four checks run on the CV-TMLE + haldensify pipeline reframe the inference:

  • ✅ Influence function mean: |mean(IF)| / SE(IF) ≈ 0 at floating-point precision. Targeting converges correctly — no implementation bug.
  • ⚠ SuperLearner meta-weights: NNLS gives HAL 0–8% weight across folds. Q is effectively XGBoost + RandomForest + Ridge; HAL is nominal.
  • ⚠ Clever covariate degeneracy: H ≡ 1/(1+δ) = 0.909 across every observation in every fold. Targeting reduces to a constant intercept shift. The CV-TMLE we ran is doubly-robust plug-in, not targeting-informed TMLE.
  • ❌ Seed sensitivity: 3 of 4 tested radii (5, 13, 19) flipped statistical significance between seed=42 and seed=1. Only R=7 is seed-robust. Per-radius significance is not publishable without cluster bootstrap.

3.1.3 Current approach: undersmoothed HAL + regHAL-TMLE

Van der Laan & Benkeser (arXiv:1908.05607, 2019) show that undersmoothed HAL on the full panel gives an asymptotically efficient plug-in estimator without requiring the targeting step that produced our degenerate clever covariate. The 20-radius sweep using this estimator with cluster bootstrap (B=50) is executing on the Lambda Vector compute node; results will replace the TMLE v3 numbers on this site when complete.

For the finite-sample inflation pattern (plug-in vs targeting), the targeted solution is the regularized HAL-TMLE (regHAL-TMLE) of Li, Qiu, Wang & van der Laan (June 2025) — Delta-method variant operates within HAL's implied working model via a ridge-regularized Newton step along the parametric efficient influence curve. Scaffold implementation is in the repo at reghal_tmle.py.

Code: tmle_core.py, undersmoothed_hal.py, reghal_tmle.py. Output: tmle_shift_365d_v3_full.csv, cv_tmle_sweep_summary.csv, hal_shift_*km.csv.

3.2. Per-well: Causal Forest DML

The per-well CATE (Conditional Average Treatment Effect) is computed by a Causal Forest (Athey, Tibshirani & Wager 2019) using the Double Machine Learning framework (Chernozhukov et al. 2018): 

Model:       econml.dml.CausalForestDML
             n_estimators=200, max_depth=4, honest=True
             Cross-fitted XGBoost nuisance models for E[Y|L] and E[A|L]

Features:    perf_depth_ft, days_active, fault_dist, fault_count,
             depth_shallow, depth_mid, depth_deep,
             neighbor_cum_vol_7km

Property:    Doubly robust — consistent if EITHER nuisance model correct
             Honest sample splitting — separate data for tree structure vs effect estimation
             Per-well 95% CIs from forest's honest variance estimate

When the user clicks an event, the dashboard:

  1. Finds all wells within R km of the event epicenter
  2. Looks up each well's features from the panel on the event date
  3. Calls CausalForestBundle.estimate_cate() which computes E[Y(A=actual) − Y(A=0) | L=l] for each well
  4. Returns per-well CATE with 95% CIs, ranked by contribution

This is an identified causal effect, not just a model prediction. Under the assumptions (no unmeasured confounding, positivity, consistency), the CATE has a causal interpretation: "how much would the expected max magnitude change if this well had not been injecting, holding confounders (including neighbor well volumes) constant?"

Code: build_causal_forest.py. Output: cf_cate_<R>km.pkl for R = 1..20 km.

3.3. Per-well threshold curve

For a specific well, the Causal Forest evaluates the CATE at a grid of hypothetical injection volumes (0 to 2.5× current volume), holding all confounders constant. This produces a dose-response curve for that one well. The regulator draws a horizontal line at their risk tolerance (e.g., CATE < 0.03 ML) and reads off the corresponding volume cap.

4. Assumptions and disclaimers

The per-well CATE numbers are identified causal effects under the stated assumptions (no unmeasured confounding, positivity, consistency). They are estimated by a Causal Forest with doubly-robust Double Machine Learning, not simple model predictions. However, they are still observational estimates with uncertainty — the 95% CIs reflect the statistical precision, and the causal interpretation relies on the assumptions below being approximately met.

The full set of assumptions:

  • No unmeasured confounding (conditional exchangeability). The DAG assumes that the listed confounders (fault distance, fault count, depth, days active, depth class, neighbor well volumes) capture all common causes of both injection volume and seismicity. Pre-existing pore pressure, fluid chemistry, fault stress state, and basement coupling depth are NOT in the panel. Sensitivity analysis (partial R², E-values) shows the effect is "VERY ROBUST" from 4 km outward.
  • Positivity / overlap. Every confounder combination must have wells at all volume levels. The TMLE dose-response grid extends to 108 BBL, but 99% of wells inject <107. Estimates above 107 are flagged as extrapolation (red shading on the chart). H-truncation at the 99th percentile (max H ≈ 5) prevents extreme density ratios from dominating the targeting step.
  • Spatial interference (partial). We include neighbor_cum_vol_7km as a confounder to partially address the SUTVA violation (wells share pressure fields). This allows estimation of "effect of well i holding nearby wells constant." However, it does not fully model spatial interference — a proper treatment would use exposure mappings and separate direct/spillover estimands.
  • No time-varying confounding model. Operators adjust injection rates in response to seismicity (feedback loop). Our cross-sectional TMLE does not model this. Longitudinal TMLE (LTMLE) would address it but requires substantial methodological development.
  • Honest estimation with sample splitting. The Causal Forest uses separate data for tree structure vs effect estimation (honest splitting). The SuperLearner uses 5-fold CV with NNLS metalearner. These prevent overfitting and produce conservative CIs.
  • Negative CATEs are real and informative. For some wells the Causal Forest estimates that injection reduced stress on this particular fault geometry. This is physically plausible (stress redistribution). Negative-CATE wells get 0% share (only positive contributions are divided across the positive total).
  • Formation data dropped. Operator-reported injection formation labels are self-reported and unreliable. Replaced with a measured depth-class proxy. Sensitivity analysis shows the indirect (pressure) pathway is robust (±7–18%); the direct effect is sensitive (±50–950% at some radii). Policy decisions should weight population dose-response curves over the NDE/NIE decomposition.
  • Wellhead pressure as mediator. WHP is a poor proxy for pore pressure at fault depth (8–10 km underground). The NDE/NIE decomposition reflects measurement limitations, not physics. The W×M interaction test (p=0.91) confirms the additive decomposition is valid, but the % mediated number should be interpreted cautiously.

5. How to read the per-well numbers

Number Interpretation
Q_factual The model's expected max ML at this well's actual feature vector on the event date. (Note: the "max ML" is the max within R km of the well, so a high Q_factual means "the model expects this well's neighborhood to have higher seismic outcomes given its feature profile" — not "this well's injection caused the event.")
Q_counterfactual The same prediction but with this well's cum_vol_365d set to 0 and BHP set to depth-only hydrostatic.
contribution Q_factual − Q_counterfactual. The model's estimate of how much of the predicted local ML goes away if you shut this well off. Can be negative.
share This well's positive contribution as a fraction of the sum of all positive contributions across the wells in the radius. Wells with negative contribution get 0%.
PN proxy Fractional reduction: 1 − Q_cf / Q_factual, clamped to [0, 1]. ≥0.5 = "more than half of the model's predicted outcome at this well would go away under shut-off." ≥0.7 = strong attribution under the model.

6. Known limitations

  • The Q is fit per-radius. The dashboard uses 20 different Q models depending on the radius slider. The same well at the same date can have a different attribution depending on the radius, because the outcome definition (max ML within R km) changes.
  • Wells outside the radius are invisible to the attribution. A well 8 km from the event with a 7 km radius contributes 0 to the in-radius decomposition, even if it's a plausible driver. Try the radius slider to see how the picture changes.
  • Multi-event days are aggregated by max ML. If two events happen on the same day at different locations, the panel records only the max nearby ML per well-day. Per-event attribution at single- event days is cleanest; multi-event days should be interpreted with extra caution.
  • Sum of contributions ≠ event magnitude. The contributions are per-well predictions from a population-fit model, not a decomposition of the event's actual ML into well shares. Don't expect Σ contribution_i to equal the observed M.
  • Leave-one-out vs Shapley. Our g-computation is "remove this one well, holding all other wells at their actual values". If there are strong interactions between wells (one well's pressure plume amplified by another's), Shapley values would give a different and more rigorous attribution. We'd need to fit Q across all subsets of nearby wells to compute true Shapley; not done.
  • Negative contributions mean what they say. If a well has contribution < 0, the model thinks shutting it off would NOT reduce expected nearby ML. This is informative — it tells the regulator that for this well, the shut-off lever wouldn't work according to the model. Don't ignore the sign.
